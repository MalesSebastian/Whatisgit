What is Git?


Git is a software that allows you to keep track of changes made to a project over time. Git works by recording the changes you make to a project, storing them in repositories, then allowing you to reference them as needed.

Basic functionalities:
  • git branch

    • A repository is structured on branches. This command can list, create or delete branches depending on your non-option arguments, if there are none of those, existing branches are listed.

    Example(deleting a branch):

    git branch -d any_local_branch

  • git checkout

    • Switches branches or restore working tree files. Specifying -b causes a new branch to be created as if git-branch were called and then checked out. git checkout commit file checks out a previous version of a file. This turns the file that resides in the working directory into an exact copy of the one from commit and adds it to the staging area.

    Example(chaging the local branch):

    git checkout any_local_branch

  • git status

    • Shows the working tree status(e.g. what files suffered changes but they are not committed).

    Example:

    git status

  • git log

    • Shows the commit logs.

    Example:

    git log

  • git push

    • Pushes the committed changes to the selected branch updating remote refs along with associated objects.

    Example:

    git push

  • git commit

    • Puts added files(git add file) located in the staging area inside the repository.

    Example(commit files added):

    git commit -m "First commit"

  • git rebase

    • Git rebase updates to the upstream head your local workspace.

    Example:

    git rebase

  • git stash

    • Saves changes on a trach of unfinished changes. Example: Often, when you have been working on part of your project, things are in a messy state and you want to switch branches for a bit to work on something else. The problem is, you don not want to do a commit of half-done work just so you can get back to this point later. The answer to this issue is the git stash command.

    Example:

    git stash

  • git reset

    • Resets current HEAD to the specified state.

    Example(Undo a commit, making it a topic branch):

    git reset --hard HEAD~3
